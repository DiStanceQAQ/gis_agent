# GIS Agent 开发任务清单（LLM Plan-and-Execute）

更新时间：2026-04-03

这份清单只保留“当前目标”任务：  
把系统落成 **LLM 主导的 Plan-and-Execute GIS Agent**，并将后端编排统一迁移到 **LangGraph**。

状态约定：

- `[x]` 已完成
- `[~]` 进行中
- `[ ]` 待完成

---

## 0. 当前目标定义

1. 任务解析、规划、推荐都由 LLM 生成结构化结果。  
2. 执行由 LangGraph 工具图驱动，并沉淀可观测事件与步骤详情。  
3. 输出处理后数据下载 / 方法说明 / 摘要，支持追问重跑。  
4. 前端是 GIS 工作台，不是聊天气泡产品。

  文件：`packages/domain/services/processing_pipeline.py`、`packages/domain/services/operation_registry.py`

  1. B档栅格操作覆盖（至少 12 项）：`raster.clip`、`raster.reproject`、`raster.resample`、`raster.mosaic`、`raster.band_math`、`raster.reclassify`、`raster.mask`、`raster.zonal_stats`、`raster.terrain_slope`、`raster.terrain_aspect`、`raster.hillshade`、`raster.rasterize`（或等价命名）。
  2. 参数可校验、错误可归类，输出包含标准元数据（CRS、分辨率、像元类型、统计摘要、nodata）。

  4. 分阶段里程碑：

     `Phase-2`：`terrain_slope/terrain_aspect/hillshade/mosaic`（已完成，支持串联执行与导出）；
     `Phase-3`：`reclassify/mask/rasterize`（已完成，支持规则重分类/几何掩膜/矢量烧录并可导出复用）。

- [x] `LPE-34` 矢量工具扩展（基础空间运算）
  文件：`packages/domain/services/processing_pipeline.py`、`packages/domain/services/aoi.py`、`packages/domain/services/operation_registry.py`
  完成标准：
  1. B档矢量操作覆盖（至少 10 项）：`vector.buffer`、`vector.clip`、`vector.intersection`、`vector.union`、`vector.dissolve`、`vector.erase`、`vector.spatial_join`、`vector.simplify`、`vector.reproject`、`vector.repair`（或等价命名）。

  3. 输出可被后续步骤复用（如 `raster.clip`、`raster.zonal_stats`、`artifact.export`），并支持矢量-栅格混合链路。
  4. 支持导出 `GeoJSON`、`GPKG`、`Shapefile`（其中 GeoJSON/GPKG 已稳定纳入回归）。
  5. 分阶段里程碑：
     `Phase-1`：`buffer/clip/intersection/dissolve/reproject`（已完成，含 GeoJSON 导出与矢量-栅格混合链路回归）；

     `Phase-3`：`repair` 与混合链路增强（已完成，支持导出 `GeoJSON/GPKG/Shapefile` 与混合链路回归）。
