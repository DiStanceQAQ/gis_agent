## 四、AOI 标准化

- [x] `AOI-01` 新建 AOI 服务模块
  文件：新建 `packages/domain/services/aoi.py`
  输出：统一入口 `normalize_aoi(...)`。
  完成标准：编排层不再直接处理 AOI 字符串或文件类型判断。

- [x] `AOI-02` 支持 `geojson / json / shp.zip` ingest
  文件：`packages/domain/services/aoi.py`、`packages/domain/services/storage.py`
  输出：从上传文件读取边界，产出统一几何对象。
  完成标准：至少支持 GeoJSON 和 `shp.zip` 两类输入。

- [x] `AOI-03` 支持 bbox 直接输入
  文件：`packages/domain/services/aoi.py`、`packages/domain/services/parser.py`
  输出：支持用户输入经纬度 bbox。
  完成标准：bbox 能转成标准 polygon 并落库到 `aois.geom / aois.bbox`。

- [x] `AOI-04` 支持地名 / 行政区解析
  文件：`packages/domain/services/aoi.py`
  输出：先做确定性行政区名解析；模糊别名仅 best-effort。
  完成标准：对“不稳定地名”返回明确确认提示，而不是静默猜测。

- [x] `AOI-05` AOI 校验与标准化
  文件：`packages/domain/services/aoi.py`、`packages/domain/models.py`
  输出：合法性检查、面积计算、bbox 生成、统一 `MULTIPOLYGON`。
  完成标准：无效几何、面积过大、空几何都能结构化报错。

- [x] `AOI-06` AOI 测试样例
  文件：新建 `tests/test_aoi.py`、新增 `tests/fixtures/aoi/*`
  输出：GeoJSON、`shp.zip`、bbox、无效多边形四类测试。
  完成标准：至少 6 条 AOI 回归样例通过。

## 七、真实 NDVI 主链路

- [x] `NDVI-01` 新建真实 pipeline 模块
  文件：新建 `packages/domain/services/ndvi_pipeline.py`
  输出：替代当前 `mock_pipeline` 的真实执行链。
  完成标准：编排层可以按开关切换 mock / real pipeline。

- [x] `NDVI-02` 影像读取与 AOI 裁窗
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：按 AOI bbox 读取窗口，不做整景全量下载。
  完成标准：小 AOI 查询不会拉整景到本地。

- [x] `NDVI-03` QA 掩膜与质量过滤
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：至少实现基础云掩膜和无效像元剔除。
  完成标准：明显高云量场景不会产出误导性结果。

- [x] `NDVI-04` 中位数合成
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：默认合成策略固定为 `median composite`。
  完成标准：时间窗内多景输入能产出单张合成结果。

- [x] `NDVI-05` NDVI 计算与统计摘要
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：计算 NDVI 栅格、最小值、最大值、均值等摘要。
  完成标准：结果摘要写入 `task_runs.result_summary_text` 对应数据结构。

- [x] `NDVI-06` PNG 渲染
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：渲染浏览用 PNG。
  完成标准：前端结果页能看到稳定预览图。

- [x] `NDVI-07` GeoTIFF / COG 导出
  文件：`packages/domain/services/ndvi_pipeline.py`
  输出：导出可下载的 GeoTIFF，必要时升级为 COG。
  完成标准：产物可在 QGIS 打开并保留地理参考。

- [x] `NDVI-08` 产物入库与下载接口对接
  文件：`packages/domain/services/orchestrator.py`、`apps/api/routers/artifacts.py`
  输出：真实产物写入 `artifacts`，前端可下载。
  完成标准：不再只依赖 mock 产物。
